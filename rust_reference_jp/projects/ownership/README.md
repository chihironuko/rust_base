# ownership  
jp reference chap4の所有権を理解するすべて  

## ルール  
- Rustの各値は所有者と呼ばれる変数と対応している  
- いかなる時も所有者は一つである  
- 所有者がスコープから外れたら、値が破棄される  

- 参照の規則  
任意のタイミングで、一つの可変参照か、複数の不変な参照のどちらかを行うことができる。  
参照は常に有効でなくてはならない。  

## 知識  
- allocateとfree  
ちょこちょこ説明で出てきた。  
ざっくり、allocate=メモリ要求/占有、free=メモリ返還

- 文字列リテラルとString  
文字列リテラルは不変であるためスタックに追加することができる。  
そのためString型より高速。  
String型は可変なため、メモリをあらかじめ多くとるのではなく、不明な量のメモリをヒープに確保する。  
そのため少し遅い。
